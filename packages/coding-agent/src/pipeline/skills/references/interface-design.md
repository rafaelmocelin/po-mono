# Interface Design

## Design It Twice

- Generate multiple radically different solutions
- Compare trade-offs
- Choose the best approach
- Implement once

## Interface Qualities

### Simplicity
- Minimal API surface
- Clear naming
- Easy to learn and use

### Generality
- Works for common cases
- Extensible for edge cases
- Avoids over-specification

### Efficiency
- Reasonable performance characteristics
- Predictable resource usage
- No hidden costs

## Design Patterns

### Layered Architecture
- Clear separation of concerns
- Each layer has defined responsibilities
- Minimize cross-layer dependencies

### Module Composition
- Small, focused modules
- Clear contracts between modules
- Pluggable implementations

## Avoiding Design Pitfalls

- Don't design for hypothetical future uses
- Don't over-generalize too early
- Don't expose internal details
- Don't make assumptions about usage
